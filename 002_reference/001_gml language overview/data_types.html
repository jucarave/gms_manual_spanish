Tipos de Datos

Esta sección cubre los diferentes tipos de datos reconocidos por GameMaker: Studio.

En secciones previas hemos visto variables y su alcance pero se ha dicho muy poco acerca de las diferencias en los tipo de datos que pueden aceptar las variables. De tal modo que este capitulo va a explicar los diferentes tipos que pueden ser almacenados y para que pueden ser usados.

Antes de continuar, vamos a explicar brevemente que son los "tipos de datos". Las variables usted crea se utilizan para guardar información, y cuando llamas una función de GML, también puede retornar información. Sin embargo esta información viene en diferentes "sabores", por ejemplo puede ser un número real o puede ser una cadena de texto. Estos diferentes tipos de valores son llamados tipo de datos, en GML existen los siguientes:

  • string - Un string es una cadena de caracteres encapsulados entre comillas sencillas ' o dobles ". Por ejemplo "The Quick Brown Fox! 123" es un string.
  • real - Un número real es simplemente un decimal o un número entero.
  • array - Un array es una variable que guarda valores usando indices consecutivos.
  • boolean - Un boolean es un valor que solo puede contener los valores true (verdadero) o false (falso).
  • pointer - Un pointer es un tipo de dato que "apunta" a un espacio de la memoria.
  • enum - Un enum es una colección de constantes que se guardan en una tipo de variable personalizada.
  • undefined - Este es un valor especial el cual es retornado cuando el valor que usted esta buscando no puede ser encontrado.

Cadenas de texto (Strings)

Un string es simplemente cualquier texto que ha sido colocado dentro de comillas (ya sean sencillas '...' o dobles "..."). Usted puede hacer ciertas operaciones con strings, por ejemplo concatenar dos strings para crear un string más largo, también puede cambiar las propiedades del string e incluso extraer números reales de ellos. Puede encontrar más información sobre estas funciones aquí: Referencia - Strings

Números Reales

Los números reales son cualquier valor que no son strings u otro tipo de dato, por ejemplo: 124, 45639.566546456, 0, -45.5, etc... Todos estos son ejemplo de números reales. Todos los números reales se guardan con el formato de punto flotante de 32bit (incluso para los valores enteros), así que puedes encontrar ciertos problemas con el redondeo de cifras. Para más información sobre esto y otras funciones relacionadas con los números, revise: Referencia - Números Reales.

Tenga en cuenta que todas las variables creadas en GameMaker se guardan como un número flotante de presición double, pero usted puede utilizar otros formatos cuando utiliza extensiones. Estas se pueden pasar a GameMaker: Studio desde una extensión y pueden ser validadas luego utilizando la función is_ indicada, la cual aparece al final de esta página.

Arrays

Un array es un tipo de dato especial el cual puede utilizarse para almacenar multiples valores. Usted asigna el array a una variable, y entonces "coloca" los valores en los diferentes indices del array. Este array puede entonces ser pasado a scripts y funciones del mismo modo que otra variable, sin embargo si usted modifica el array dentro de la función esta se convertira en una copia del array original y entonces necesita ser retornado a la variable del array original para que los valores puedan ser actualizados. Para más información sobre los arrays, revise Revisión de GML - Arrays.


Booleans

Un booleano es simplemente un valor que puede ser true (verdadero) o false (falso). Tenga en cuenta que GameMaker: Studio no soporta "verdaderos" valores booleanos, y acepta cualquier número por debajo de 0.5 como false y cualquier número mayor o igual a 0.5 como verdadero. Esto no signinifca que debas utilizar 1 y 0 (u otro número real) para verificar variables true y false, y que GameMaker: Studio tiene las constantes true y false las cuales deberían ser utilizadas siempre en su código para evitar cualquier problema, en una actualización futura se añadiran verdaderos valores booleanos como tipo de datos.

Punteros

Un puntero es un tipo de dato que "apunta" a un lugar en la memoria. Usted no puede hacer operaciones con un puntero y es utilizado unicamente para algunas funciones especificas, como por ejemplo, obtener una textura o la dirección de memoria de un buffer. Para conocer las funciones que retornan punteros revise: buffer_get_address o sprite_get_texture.

También hay una función para revisar si algún valor es un puntero (Revise "Validando tipos de dato", abajo) y una función para convertir un valor a un puntero:

  1. ptr

Enums

Un enum es una "enumeración", y básicamente te permite crear tu propio tipo de dato de valores constantes. Los Enums son variables de alcance global (esto es de forma implicita, no requiere el sufijo "global" o globalvar) y tienen la siguiente estructura:

enum <variable>{<constant> [= <value>]}

En el siguiente ejemplo, creamos una enumeración que contiene los colores del arcoiris y asigna diferentes constantes como sus valores:

enum rainbowcolors {
   red,
   orange,
   yellow,
   green,
   blue,
   indigo,
   violet
   }

Los valores de los enum solamente pueden ser números reales o expresiones con otros enums, y por defecto estan enumerados desde 0 en adelante, así que para nuestro ejemplo anterior los valores serían: red = 0, orange = 1, yellow = 2, etc... Usted también puede asignar los valores a un enum manualmente al momento de definirlos, usando números reales o expresiones:

enum rainbowcolors {
   red = 5,
   orange = 5 * 2,
   yellow = 15,
   green = 20,
   blue = 25,
   indigo = 30,
   violet = 35 * enum_test.entry
   }

Notese que en el anterior ejemplo utilizamos otro enum para crear la expresión "violet". Esto solo funcionará si el enum siendo referenciado fue creado previamente, esto es porque el valor del enum debe ser evaluado como una constante al compilar.

Para acceder luego a valores que existen dentro de un enum, usted puede utilizar el método del punto ".", de la siguiente forma:

var value = rainbowcolours.green;

Tenga en cuenta que usted no puede modificar ningún valor de los enums luego de que estos han sido creados.

Undefined

Un valor undefined (indefinido, también conocido como "null") es uno donde una expresión no tiene un valor correcto, sin embargo como es sintácticamente correcto, debe retornar algo. Por ejemplo, supongamos que usted tiene un ds_map y utiliza la función ds_map_find_value(). Ahora, ¿qué pasa cuando el mapa no contiene el valor buscado? como la función esta bien formada, y el problema es que el valor no existe, entonces retornaría la constante undefined, y usted puede validar esta constante del mismo modo que lo haría con true o cualquier otro valor.

Validando tipos de dato

GameMaker: Studio le permite validar que el tipo de dato de alguna variable sea de un tipo conocido con las siguientes funciones:

  1. is_string
  2. is_real
  3. is_array
  4. is_ptr
  5. is_int32
  6. is_int64
  7. is_matrix
  8. is_undefined
