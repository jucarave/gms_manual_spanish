Descripción del GML

GameMaker: Studio tiene su propio lenguaje de programación llamado GameMaker Language (Abreviado a GML).

El lenguajde de programación de GameMaker: Studio, le otorga mucha más flexibilidad y control que las acciones básicas que están disponibles a traves de la interfaz Drag'n'Drop (DnD). Hay un grán número de lugares en los que se puede sar este lenguaje para crear y controlar su juego:

  • Scripts. Un script es (usualmente) pequeños pedazos de código GML que usted escribe para crear sus propias funciones.

  • Eventos. Usted puede usar código dentro de los eventos para controlar el comportamiento y las interacciones de las instancias de esos objetos. ¡Se puede mezclar incluso Drag'n'Drop con código!

  • Código de Creación de Rooms. Cada room tiene un "evento create" específico para esa room. Aquí usted puede añadir el código que necesite y se ejecutara cada vez que se entra en el room.

  • Código de Creación de Instancias Del mismo modo que se puede insertar dentro de los eventos de un objeto, usted también puede añadir código a instancias individuales cuando las ubica en los rooms. Este código se ejecutara despues del evento de creación del objeto y es específico a la instancia a la que se añada. Así que puedes tener cuatro instancias del mismo objeto, cada uno con propiedades diferentes gracias a este código.

En esta sección vamos a describir la estructura básica de las funciones GML y como pueden ser usados para crear sus propios programas y generalmente controlar su juego. Tenga en cuenta que antes de programar en GameMaker: Studio ¡hay algunas convenciones que usted debería seguir! Primero que todo, para todos los recursos (sprites, objetos, sonidos, etc...) que usted utilice deben tener nombres que comiencen con una letra y que solo contengan letras, dígitos y el simbolo de guión bajo '_' (por ejemplo "spr_Fish" es aceptable pero no lo es "spr Fish" o "6Fish") de otro modo no podrá hacer referencia a ellos desde las funciones GML. También, asegúrese de que todos sus recursos tengan un nombre diferente (Hay una función en la sección de Scripts del editor principal que puede revisar esto por usted), y no nombre ningún recurso como self, other, global, o todos ya que estos tienen un significado especial en el lenguaje.
